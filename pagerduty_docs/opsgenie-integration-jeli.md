Opsgenie Integration | Jeli
===========================

Jeli Part of PagerDuty

Opsgenie + Jeli Benefits
========================

* Pull in responders during an incident
* Use Opsgenie data during analysis to see who involved was on call

Requirements
============

> 📘 Availability
> --------------
> 
> Jeli is available to customers with a standalone Jeli plan, or if they are on the Enterprise Incident Management or Enterprise Customer Service Ops plans. For customers who have Jeli as part of their pricing plan, all users (regardless of [user role](/main/docs/advanced-permissions#role-types)) will have access to Jeli. Please contact our [Sales Team](https://www.pagerduty.com/contact-us/#contact-sales) to upgrade to an account plan with Jeli.
> 
> At this time, Jeli is not available for accounts in the EU [service region](/main/docs/service-regions) (i.e., `https://subdomain.eu.pagerduty.com`).

  

To Configure the Integration
----------------------------

> 🚧 Integration Requirement
> -------------------------
> 
> A Jeli account can only have an Opsgenie or [PagerDuty](/main/docs/pagerduty-integration-jeli) integration configured, not both.

### In Jeli

Admin permissions are required to manage Jeli Integrations.

### In Opsgenie

Admin rights are required to create Opsgenie API keys.

How it Works
============

* Via the [Jeli Slackbot](/main/docs/slack-integration-jeli), users initiate and/or escalate an Opsgenie incident.
* In Slack, users can view and follow links to the incident in Opsgenie.

Integration Walkthrough
=======================

I. In Opsgenie
--------------

### Create an Opsgenie API Key

1. Navigate to **Settings** and select **API key management** in the left navigation menu.
2. On the page that appears, click **Add new API key**.
   
   ![An image of the Opsgenie web app detailing how to add a new API key](https://files.readme.io/f2db0a6-jeli-opsgenie-api-key-management.png)
   
   
   
   Add a new Opsgenie API key
3. In the modal, enter a **Name** for your API key. We recommend using a name related to Jeli, e.g., `jeli-opsgenie-integration`.
4. Under **Access rights**, check the following boxes:
   
   1. **Read**
   2. **Create and update**
   3. **Delete**
   4. **Configuration access**
5. **Copy** the generated key to your clipboard.
6. Click **Add API key**.

II. In Jeli
-----------

### Add an Opsgenie API Key

1. Navigate to **Settings**  **Integrations**  **Opsgenie**.
2. Click **Connect Opsgenie**.
3. In the **Account API key** field, paste the API key from the [previous section](#create-an-opsgenie-api-key) and click **Submit**.

III. In Opsgenie
----------------

> 🚧 Integration Keys
> ------------------
> 
> Integration keys allow you to page to an Opsgenie team using the [Jeli Slackbot](/main/docs/slack-integration-jeli). If you’re on a non-enterprise tier, Opsgenie requires you to create an integration key for each team.

1. In the top navigation, select **Teams** to find your integrations.
2. Select the **Team** you’d like to create an integration key for.
3. In the left navigation menu, click **Integrations**.
4. Perform the following based on your use case:
   1. **If you have an existing integration**: Select the **integration** from the list, and on the next page copy the **API key** under the Integration settings section.
   2. **If you are adding a new integration**:
      1. Click **Add Integration** in the upper right.
      2. Select the **API** integration in the **All Products** section.
      3. Enter an **Integration name**, select an **Assignee team**, and click **Continue**.
      4. Find the newly created integration and copy its **API key** as noted [above](#create-an-opsgenie-api-key) .

IV. In Jeli
-----------

You can add multiple Opsgenie integration keys, depending on your Opsgenie account plan. For example, OpsGenie Enterprise plans can use one integration key for all teams, but if you wish to page through `/jeli escalate` in Slack and are on a lower pricing tier in Opsgenie, they will require a separate integration key per team.

1. In Jeli, navigate to **Settings**  **Integrations**  **Opsgenie**.
2. Click **Connect Opsgenie**.
3. Make sure you’ve [added your Account API key](#create-an-opsgenie-api-key). Under **Account API Key**, you should see the domain you’ve connected, a date, and by whom.
4. In the section **Integration API Key**, click **Add integration key**.
5. In the row that appears, add the **Integration Key Name**, which must exactly match the integration key name in Opsgenie (e.g., `Engineering_API`), and paste the integration key in the **API key** field. If you have multiple integration keys, you can add them all at once by clicking **Add integration key**.

![A screenshot of the Jeli UI detailing how to add an Opsgenie integration key to Jeli](https://files.readme.io/3e898333f4cedc9eb0c84eb9f3ae6d1bd3d0775c5e95a98cf2d72f7ce2476418-integration_key_name.png)



Add an Opsgenie integration key to Jeli



6. Click **Connect account** to save.

Opsgenie | Jeli Integration User Guide
======================================

Create an Opsgenie Alert
------------------------

1. Once configured, during an active incident channel you can run either `/jeli page` or `/jeli escalate` to create an Opsgenie alert.
2. Select the team in Opsgenie you would like to page, and *optionally* customize the message to add context as needed.
3. Click **Create Alert**.
